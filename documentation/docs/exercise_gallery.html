LMS Exercise Types

A comprehensive showcase of all exercise types available in our Learning Management System

Overview

The LMS currently supports 10 different exercise types, each designed for specific learning scenarios and objectives. This index provides an overview of all available exercise types, their features, and use cases.

All exercise types can be seen in action on our Exercise Showcase pages:

Complete Exercise Showcase Simplified Showcase

Word Scramble

Type ID: word_scramble

Students rearrange words to form a complete, meaningful sentence.

Features:

  • Drag-and-drop interface
  • Automatic scoring
  • Visual feedback for correct ordering
Example: Arrange the words: "quick brown the fox jumps"
Grammar Sentence Structure Language Syntax

Multiple Choice

Type ID: multiple_choice

Students select one correct option from several choices.

Features:

  • Single-selection radio buttons
  • Automatic grading
  • Support for image options
Example: What is the capital of France?
A) London B) Berlin C) Paris D) Madrid
Factual Knowledge Concepts Comprehension

Short Answer

Type ID: short_answer

Students provide a concise answer (typically a few words or a sentence).

Features:

  • Text input field
  • Word count limit option
  • Automatic or manual grading
Example: Define photosynthesis in one sentence.
Definitions Recall Brief Explanations

Long Answer

Type ID: long_answer

Students write extended responses, essays, or paragraphs.

Features:

  • Rich text editor
  • Word count min/max limits
  • Manual grading with rubric
Example: Explain the causes and effects of climate change (min 200 words).
Critical Thinking Analytical Skills Detailed Explanations

True/False

Type ID: true_false

Students determine whether a statement is true or false.

Features:

  • Simple binary choice
  • Automatic grading
  • Optional explanation field
Example: The sun revolves around the Earth. (True/False)
Factual Knowledge Quick Assessment Understanding

Fill in the Blanks

Type ID: fill_blank

Students complete sentences by filling in missing words.

Features:

  • Text input fields inline with content
  • Multiple blanks in a single exercise
  • Automatic grading with exact or fuzzy matching
Example: The process of plants making food using sunlight is called ________.
Vocabulary Grammar Knowledge Points

Matching Words

Type ID: matching_words

Students match items from two columns (e.g., terms with definitions).

Features:

  • Drag-and-drop or dropdown selection
  • One-to-one correspondence
  • Automatic grading
Example: Match the countries with their capitals.
Relationships Terms & Definitions Cause & Effect

Image Labeling

Type ID: image_labeling

Students label specific parts of an image.

Features:

  • Interactive image with clickable points
  • Drag labels to locations or dropdown selection
  • Automatic grading based on label placement
Example: Label the parts of the human heart in this diagram.
Anatomy Geography Visual Comprehension

Sentence Reordering

Type ID: sentence_reordering

Students arrange sentences in the correct logical order.

Features:

  • Drag-and-drop interface for reordering
  • Automatic sequence checking
  • Support for paragraph reconstruction
Example: Arrange these sentences to form a coherent paragraph about the water cycle.
Narrative Comprehension Logical Sequencing Procedural Understanding

Cloze Test with Word Bank

Type ID: cloze_test

Students fill in blanks in a text using words from a provided word bank.

Features:

  • Word bank with drag-and-drop functionality
  • Multiple blanks in a passage
  • Automatic grading
Example: Complete the passage using words from the word bank: climate, atmosphere, temperature, greenhouse.
Vocabulary in Context Reading Comprehension Grammar

Technical Implementation

All exercise types are implemented as React components in the frontend/app/components/exercises directory. The central dispatcher for all exercise types is the ExerciseRenderer component, which renders the appropriate exercise component based on the exercise_type property.

In the backend, each exercise type has a corresponding schema definition in backend/app/schemas.py, and the database uses a flexible JSON-based approach to store exercise-specific data.
